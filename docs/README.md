# `docs/` 项目文档

本目录保存会长期影响产品与后续协作的项目结论。

## 核心文档

- `DECISIONS.md`：已确认的产品、页面边界、结构和技术结论。
- `DESIGN.md`：已确认的长期视觉、交互和中文阅读规则。
- `ROADMAP.md`：当前、下一步和暂缓事项。

## 保存原则

- 日常执行计划在开始工作前向用户展示，不默认写入仓库。
- 常规实现结果由 Git 历史和最终回复记录，不重复创建阶段结果文档。
- 参考资料不默认复制进仓库；需要长期保存时记录来源，并避免敏感材料和受版权保护的完整内容。
- 只有用户明确要求长期保存时，才在 `docs/` 中创建命名清晰的文件，并在本页补充导航；不为停用的文档类型保留空目录。
