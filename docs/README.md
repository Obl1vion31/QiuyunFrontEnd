# `docs/` 项目文档

本目录保存会长期影响产品与后续协作的项目结论。

## 核心文档

- `DECISIONS.md`：已确认的产品、页面边界、结构和技术结论。
- `DESIGN.md`：已确认的长期视觉、交互和中文阅读规则。
- `ROADMAP.md`：当前、下一步和暂缓事项。

## 历史目录

- `exec-plans/`：默认不再创建阶段执行计划。
- `exec-results/`：默认不再创建阶段执行结果。
- `references/`：默认不维护 Reference 生命周期文档。

只有用户明确要求长期保存时，才重新使用历史目录。日常复杂任务可以在执行前展示临时计划，不默认写入仓库。
