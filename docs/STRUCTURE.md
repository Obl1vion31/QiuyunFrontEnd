# 信息架构

本文件用于记录页面结构、导航关系、模块关系和用户路径。

## 总枢纽首页的结构定位

总枢纽首页是系统入口，不是整个前端团队管理系统。

它主要负责：

- 说明前端团队为什么存在；
- 展示前端业务的完整工作链路；
- 表明不同岗位在链路中的协作关系；
- 提供业务、组织人员、绩效报酬等系统入口；
- 帮助用户进入后续业务、人员或绩效相关系统。

它不应直接承载：

- 周报；
- 月报；
- 实时数据看板；
- 当前项目列表；
- 本周风险面板；
- 完整业务排期表；
- 所有 SOP、规则和模板正文；
- 整个前端系统的全部内容。

详细业务内容应进入相应业务模块，不应持续堆积在总枢纽首页。

## 初始模块边界

当前可按以下高层区域理解系统结构：

- 总枢纽：解释系统和提供入口；
- 业务系统：承载计划、排期、任务、数据、复盘和问题跟踪；
- SOP、规则与模板：承载可复用方法和规范；
- 组织与人员：承载岗位、人员、培训和晋升相关内容，是否进入网站仍待判断；
- 绩效与报酬：承载敏感管理内容，是否进入网站仍待判断；
- 外部智能文档：作为过渡资料和外部入口存在。

上述结构不是最终导航方案。正式页面、导航层级和模块优先级仍需后续确认。

## Phase 01 结构基线

Phase 01 先实现一个可扩展的总枢纽首页，不建设完整业务系统。

第一版页面结构应采用连续纵向页面：

1. Hero 与页面定位；
2. 前端团队为什么存在；
3. 四个详细目标；
4. 前端整体协作链路；
5. 链路执行底座；
6. 岗位在链路中的位置；
7. 三个系统入口。

内容来源：

- `docs/references/Hero_Ref.md` 只作为首页应包含内容的范围依据；
- 不直接复刻该 Markdown 文件的格式、标题层级或列表样式；
- 首页内容应拆成可维护配置，页面组件负责表达结构和视觉层级。
- 首页核心主旨、精简总结路径和四个详细目标是三个不同层级，不应互相替换。
- 系统入口是首页最后一个主要内容区块，页面不再展示使用路径说明或维护边界说明。

系统入口基线：

- 业务系统：Phase 01 可显示外链、占位或建设中状态；
- 组织与人员：Phase 01 可显示外链、占位或建设中状态；
- 绩效与报酬：Phase 01 只显示授权入口概念，不实现登录和权限。

未来扩展位置：

- 业务计划、排期、项目和数据复盘应进入业务系统；
- 岗位、培训、职级和 SOP 可进入组织与人员系统或外部智能文档；
- 绩效与报酬内容需等待权限和敏感信息边界确认。
