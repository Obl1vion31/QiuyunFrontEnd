# 信息架构

本文件用于记录页面结构、导航关系、模块关系和用户路径。

## 总枢纽首页的结构定位

总枢纽首页是系统入口，不是整个前端团队管理系统。

它主要负责：

- 说明前端团队为什么存在；
- 展示前端业务的完整工作链路；
- 表明不同岗位在链路中的协作关系；
- 提供业务、组织人员、绩效报酬等系统入口；
- 帮助不同用户找到后续要进入的具体模块或文档。

它不应直接承载：

- 周报；
- 月报；
- 实时数据看板；
- 当前项目列表；
- 本周风险面板；
- 完整业务排期表；
- 所有 SOP、规则和模板正文；
- 整个前端系统的全部内容。

详细业务内容应进入相应业务模块，不应持续堆积在总枢纽首页。

## 初始模块边界

当前可按以下高层区域理解系统结构：

- 总枢纽：解释系统和提供入口；
- 业务系统：承载计划、排期、任务、数据、复盘和问题跟踪；
- SOP、规则与模板：承载可复用方法和规范；
- 组织与人员：承载岗位、人员、培训和晋升相关内容，是否进入网站仍待判断；
- 绩效与报酬：承载敏感管理内容，是否进入网站仍待判断；
- 外部智能文档：作为过渡资料和外部入口存在。

上述结构不是最终导航方案。正式页面、导航层级和模块优先级仍需后续确认。
