# 已确认决定

本文件只记录已经确认、会影响后续工作的产品、结构和技术结论。

## 产品定位

- 当前产品暂称“前端团队管理系统”。
- 网站长期目标是承载前端业务工作、数据、规则、复盘和协作入口，不应停留在静态宣传页。
- 首页以部门工作流为主要叙事，呈现策划、内容、视觉、发布、用户触点和反馈回流如何组成完整流程；对外文案不强调个人或当前人员安排。
- 总枢纽首页是系统入口，不是整个系统，也不是实时业务驾驶舱。
- 前端负责从用户看见内容，到阅读、产生兴趣、形成咨询并完成有效交接的过程；最终销售成交仍属于销售侧边界。
- 原始业务数据不应主要存放在周报或月报中；报告是输出视图，不是事实来源。

## 当前页面边界

- 当前正式界面只有总枢纽首页。
- 首页用于解释前端存在的意义、完整工作链路和后续系统入口。
- 首页首屏同时呈现工作台定位与前端使命：以 `FRONTEND CONTENT WORKSPACE` 说明平台属性，以 `WHY WE EXIST` 说明内容触达目标；不再设置独立的使命区块。
- 首页不展示实时数据、周报明细、月报明细、当前项目列表、完整业务排期或个人绩效薪资明细。
- 首页当前保留业务系统、组织与人员、绩效与报酬三个方向的入口概念。
- 入口可以处于占位、外链或授权访问状态；当前不实现登录和权限。
- 未来业务页面、路由和字段在进入对应阶段时再确认，不提前设计完整系统蓝图。

## 技术基线

- 技术栈采用 Astro 4 + React 18 + TypeScript，以兼容本地 Node `18.19.1`。
- 包管理器采用 `pnpm`。
- 样式以 Astro scoped CSS 为主，复杂交互组件可以使用 React 和 CSS Modules。
- 首页内容集中维护在 TypeScript 配置中，避免正文散落在组件里。
- 当前只做本地验收，不配置生产部署。
- 当前不建设数据库、后台、登录权限、自动采集或报告生成。

## 文档基线

- 仓库主要内容目录维护各自的 `README.md`，用于说明目录用途、文件职责和安全边界；`src/pages/` 除外，避免 Astro 把说明文档生成为新路由。
- `docs/` 使用 `README.md` 导航，长期结论仍只进入 `DECISIONS.md`、`DESIGN.md` 和 `ROADMAP.md`。
- 产品、结构和技术的已确认结论统一进入本文件。
- 视觉与交互规则只在方向获得确认后进入 `DESIGN.md`。
- 当前工作、下一步与暂缓事项进入 `ROADMAP.md`。
- 不再常规维护独立的产品定义、结构说明、执行计划、执行结果和 Reference 生命周期文档；历史由 Git 保存。

## 当前视觉状态

- 原首页视觉和布局已被“内容穿越总控台”方向替代。
- 首页是全站唯一允许大胆视觉表达的页面；后续页面和首页后半段保持简洁、清晰和低动效。
- 首页 Hero 使用大量 3:4 帖子封面建立内容纵深，滚动时封面从远处向前穿过工作台。
- 六条轨道表达策划、内容、视觉、发布、用户触点和复盘依次衔接并持续回流的部门工作流。
- 必须保留前端宗旨：让用户愿意点击、愿意读且读完，并愿意进一步了解或咨询。
- 当前没有真实帖子封面；`imagegen` 因网络错误未生成输出，页面暂时使用可替换的代码封面板，不把它们视为真实业务素材。
