# 设计方向

## 总体原则

- 首页是全站唯一允许大胆视觉表达的页面。
- 首页后半段和未来业务页面保持简洁、低动效和高可读性。
- 视觉必须服务真实前端工作：内容策划、阅读体验、视觉设计、发布、用户触点和复盘。
- 不用普通卡片网格、装饰性编号或无业务含义的渐变制造“设计感”。
- 中文阅读体验、移动端可用性、键盘焦点和减少动态效果模式属于质量底线。

## 首页概念：内容穿越总控台

首页的标志性场景是一张呈现完整前端工作流的内容工作台。

- 大量 3:4 帖子封面分布在不同深度、位置和角度。
- 用户正常滚动页面时，封面从远处向前靠近并从视野边缘离开。
- 封面的速度、位置和旋转可以不规则，但不能遮挡核心宗旨文字。
- 同时只允许少量封面完全清晰；远处封面降低亮度并增加轻微模糊。
- 六条轨道表达策划、内容、视觉、发布、用户触点和复盘如何依次衔接并持续回流。
- 大胆程度集中在这个场景，离开 Hero 后停止 3D 穿越效果。

## 必须保留的内容宗旨

- 让用户愿意点击；
- 让用户愿意读，且愿意读完；
- 让用户愿意进一步了解与咨询。

宗旨必须使用网页文字呈现，不写入生成图片，保证准确、清晰和可访问。

## 视觉变量

- `Control Night #10243E`：Hero 与工作轨道背景；
- `Signal White #F2F4EF`：页面底色和主要文字；
- `Blueprint #2454D3`：轨道、标签和结构信息；
- `Marker Orange #FF6A3D`：当前流程环节和关键动作；
- `Monitor Mint #A9DDC8`：少量辅助状态。

标题使用紧凑、强重量、较窄的无衬线表达；正文使用适合中文连续阅读的中性无衬线；状态和编号使用等宽字体。未引入外部字体依赖，正式品牌字体后续单独确认。

## 帖子封面

- 正式素材优先使用真实小红书帖子封面。
- 推荐比例为 3:4；组件也以 `aspect-ratio: 3 / 4` 约束。
- 素材应先移除不应公开的信息，不编造业务结果或用户数据。
- 当前临时封面板仅用于验证空间和交互，不属于正式内容资产。
- 封面配置支持后续添加真实 `image` 路径，不应为了替换素材重写动画。

## 动效与性能

- 滚轮继续控制正常页面进度，不锁定滚动，不制造强制翻页。
- 使用 CSS 3D 与轻量 React Island 映射滚动进度，不引入大型 3D 库。
- 活跃封面数量保持有限，通过不同深度和循环形成丰富感。
- 支持 `prefers-reduced-motion`；减少动态效果时保留静态纵深，不持续推进封面。
- 移动端降低封面尺寸、运动强度和轨道密度。
