# 设计方向

本文件用于记录长期稳定的视觉和交互原则，不用于保存单个页面的完整视觉稿或临时设计过程。

## 基础原则

产品主要界面语言为简体中文。

页面应：

- 保持清晰的信息层级；
- 保证中文阅读自然；
- 同时适配桌面端和移动端；
- 使用语义化结构；
- 避免不必要的装饰；
- 避免过度动画；
- 避免将网页做成多页 PPT；
- 避免重复解释同一件事。

## 产品内的视觉关系

首页、业务系统和数据看板应共享一套视觉语言，但信息密度可以不同：

- 总枢纽首页偏解释、导航和系统入口；
- 业务系统偏操作、筛选、状态和任务推进；
- 数据看板偏扫描、比较、判断和异常识别；
- 报告页面偏结论、证据、趋势和决策追踪。

## 当前阶段先确定的内容

- 中文阅读体验优先；
- 信息层级要清晰；
- 总枢纽首页保持结构性和导航性；
- 业务数据和实时状态不堆放在首页；
- 设计风格应服务团队管理和业务判断，不应过度装饰。

## 留到页面实施阶段确定的内容

- 具体颜色系统；
- 字体和字号比例；
- 首页最终视觉方案；
- 组件细节；
- 动效细节；
- 图表样式；
- 单个模块的页面布局。

## Phase 01 设计基线

Phase 01 首页应形成适合网站连续阅读的视觉表达，而不是复刻 Markdown 文档或 PDF/PPT 页面。

已确认原则：

- `docs/references/Hero_Ref.md` 只定义首页包含什么内容，不定义最终样式。
- 页面应有清晰的中文标题、说明、段落和入口层级。
- 首页可以保留深色 Hero、暖色强调、克制安静的方向感，但不要求机械复刻 PDF 背景或 Markdown 排版。
- 协作链路应优先保证可读性，桌面端可以更有结构感，移动端必须单列清晰。
- 卡片用于入口、目标、角色和信息组，不做无意义装饰。
- 视觉效果应服务“理解前端团队如何运作”和“找到系统入口”。

Phase 01 不建立完整设计系统，但实现时需要沉淀可复用的基础变量：

- 页面最大宽度；
- 基础间距；
- 主要文字层级；
- 强调色；
- 深色与浅色区域的使用原则；
- 移动端断点。
