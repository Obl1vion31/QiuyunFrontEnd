# 执行计划

本目录用于记录具体任务的执行计划。

路线图用于说明项目阶段和优先级；执行计划用于说明某一次具体任务的目标、范围、步骤、验证方式和结果。

## 使用原则

- 一个较大的任务可以对应一个独立执行计划文件。
- 执行计划应聚焦具体任务，不替代产品定义或路线图。
- 执行计划中可以记录假设，但正式决定应沉淀到 `docs/DECISIONS.md`。
- 执行计划完成后，应记录实际完成内容和验证结果。

## 计划内容建议

复杂任务的执行计划至少说明：

- 任务目标；
- 范围；
- 非目标；
- 假设；
- 开放问题；
- 实施步骤；
- 验收标准；
- 测试方式；
- 风险；
- 回滚方式。

## 验证内容建议

代码实施类任务完成后，应根据项目已经配置的工具执行检查。常见检查包括：

- 格式化检查；
- lint；
- 类型检查；
- 自动化测试；
- 生产构建；
- 桌面端页面检查；
- 移动端页面检查；
- 链接和资源检查；
- 控制台错误检查。

如果某项检查无法执行，必须说明原因，不得声称已经通过。
